# Poet Project Guidelines

## Project Structure

This project follows a specific structure that should be maintained:

- `src/`: Source code
  - `datasets/`: Dataset implementations
  - `loaders/`: Checkpoint loaders
  - `poet/`: Model definitions
  - `trainers/`: Training logic
  - `utils/`: Utility functions
- `scripts/`: Training and utility scripts
- `configs/`: Configuration files (YAML)
- `test/`: Test scripts
- `data/`: Dataset files (gitignored)

## Development Guidelines

### Testing

- All test scripts should be placed in the `test/` directory
- Test files should be named `test_*.py` for consistency
- Tests should verify functionality before full training runs

### Dataset Configuration

- Use `local_imagenet` for local parquet files
- Use `imagenet` for WebDataset format
- Dataset path should point to the directory containing parquet files
- Cache directory should be `data/.cache` for processed data

### Code Style

- Use logger instead of print for all logging
- Logger should be passed from training scripts, not created internally
- All dataset classes should accept an optional `logger` parameter
- Use `data_path` consistently (not `dataset_path` or `root_path`)

### Import Conventions

- Use relative imports within packages (e.g., `from .base_loader import`)
- Absolute imports from other packages (e.g., `from utils.logger import`)
- All package `__init__.py` files should use relative imports
